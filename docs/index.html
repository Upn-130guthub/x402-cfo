npm install x402-cfo View on GitHub
agent.ts
import { AgentCFO, JsonFileStorage } from 'x402-cfo';

const agent = new AgentCFO({
  wallet: myX402Wallet,
  budget: { hourly: 5, daily: 50, session: 200 },
  policy: { maxPerRequest: 2.00, allowedCurrencies: ['USDC'] },
  storage: new JsonFileStorage('./ledger.json'), // Survive restarts
});

// React to financial events
agent.events.on('budget:warning', ({ window, percentUsed }) => {
  console.warn(`⚠️ ${window} budget at ${(percentUsed * 100).toFixed(0)}%`);
});
agent.events.on('velocity:spike', ({ multiplier }) => {
  console.warn(`🔥 Spending ${multiplier.toFixed(1)}x above average`);
});

// Drop-in fetch — auto-handles 402 → policy → budget → pay → log
const res = await agent.fetch('https://api.chaindata.xyz/v1/prices');

agent.estimateCost('https://api.chaindata.xyz/v1/prices');
// → { average: 0.25, min: 0.20, max: 0.35, samples: 47 }

Features

Financial control for autonomous agents.

Zero runtime dependencies. Everything your agent needs to spend responsibly.

🏦
Budget Enforcement
Per-request, hourly, daily, and session spend limits with rolling-window tracking.
📋
Cost Policies
Declarative rules: URL allowlists, blocklists, currency restrictions, and network filters.
📒
Audit Ledger
Every payment decision logged with timestamp, amount, URL, status, and reason. Exportable.
⚡
Event System
Typed events for payment success, denial, budget warnings, and velocity spikes. Wire up Slack alerts or circuit breakers.
💾
Persistent Storage
Ledger survives restarts. Ships with JsonFileStorage, pluggable StorageAdapter for SQLite or Redis.
🔮
Cost Estimation
Learns what endpoints cost from history. Query average, min, max before making a call.
🔥
Velocity Detection
Fires an event when spending 2x+ above the historical average. Catch runaway agents before they burn budget.
📊
Spend Analytics
Burn rate, projected daily spend, top endpoints by cost, and currency breakdown.
🖥️
Express Dashboard
One-line middleware for Express. Includes a live dashboard at /_cfo/html with spend cards, transaction table, and auto-refresh.
